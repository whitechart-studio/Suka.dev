# Team Connection and Hosted Mode

Suka starts local-first, but the product direction includes hosted and self-hosted team workspaces.

## Team Connection UI

The dashboard should include a clear team connection surface:

- Local-only mode indicator.
- Team-connected mode indicator.
- Invite link or token workflow.
- Connected teammates and agents.
- Connection health.

## Hosted Mode Goals

- Team workspaces.
- Project IDs.
- Agent tokens.
- Authenticated dashboard access.
- Audit logs.
- Retention controls.
- Privacy controls.

## Self-Hosted Mode Goals

- Docker-based deployment path.
- Configurable persistence.
- Clear token management.
- No forced hosted dependency.

## Privacy Position

Suka should avoid uploading source code by default. Coordination metadata should be explicit and configurable.
